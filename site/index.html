ClaudioOS

No Linux. No POSIX. No JavaScript.
Just Rust, UEFI, and Claude.

Phase 1: Boot to Terminal Rust nightly x86_64 AGPL-3.0-or-later

A bare-metal operating system purpose-built for running multiple Anthropic Claude coding agents simultaneously. Boots via UEFI, speaks HTTP/1.1 + TLS directly from Ring 0, renders a split-pane terminal on the GOP framebuffer.

// Features

Networking

Native VirtIO + smoltcp

Custom VirtIO-net driver with DMA virtqueues. smoltcp provides TCP/IP, DHCP, and DNS. No Linux networking stack.

Security

Bare-Metal TLS

TLS handshake and encrypted streams from Ring 0. HTTPS POST directly to api.anthropic.com with no OS intermediary.

Display

Split-Pane Terminal

Binary split tree layout over the GOP framebuffer. VTE parser for full ANSI escape sequences. Truecolor support.

AI

Anthropic Tool Use

SSE streaming parser for Claude's Messages API. Tool use protocol with file I/O on FAT32. Multi-agent sessions.

Auth

OAuth Device Flow

RFC 8628 device authorization at boot. Display code on framebuffer, authorize from your phone. Token refresh in background.

Runtime

Async Executor

Interrupt-driven cooperative scheduler. Hardware IRQs wake futures. HLT when idle. No polling, no wasted cycles.

// Boot Sequence

Phase -1

Proof of Life

Raw serial write to port 0x3F8. Bytes hit the wire before anything else initializes.

Phase 0

Serial + Logger

16550 UART initialization (115200 8N1). Log crate wired to serial output.

Phase 1

GDT + TSS

Global Descriptor Table with kernel code, data, and TSS segments. IST stacks for double-fault and timer handlers.

Phase 2

Memory

OffsetPageTable from CR3. 1 MiB heap mapped at 0x4444_4444_0000. linked_list_allocator as global allocator.

Phase 3

Interrupts

APIC disabled. 8259 PIC initialized (ICW1-4). IDT loaded. IRQ0 (timer) + IRQ1 (keyboard) unmasked. STI deferred.

Phase 4

Framebuffer

GOP framebuffer physical address resolved via page table walk. Writable access through phys_mem_offset mapping.

Phase 5

PCI Enumeration

Bus 0 scan. VirtIO-net, e1000 detected. Bus mastering enabled for DMA devices.

Phase 6

Stack Switch + Executor

256 KiB heap stack allocated and leaked. RSP switched via inline ASM. Interrupts enabled. Executor runs forever.

// Design Decisions

No JavaScript Runtime

Call the Anthropic API directly via raw HTTP/1.1 + SSE over TLS. Eliminates 50+ syscalls worth of compatibility shims.

Single Address Space

No kernel/user boundary. No syscalls. No process isolation. Every agent session is an async task. We trust our own code.

Interrupt-Driven Async

Hardware interrupts (NIC RX, keyboard, timer) wake futures. HLT when idle for power savings. No busy-loop polling.

FAT32 Persistence

No ext4, no journaling. Tokens, config, conversation logs, and agent state persisted to a simple FAT32 partition.

// Build & Run

# Build the kernel
cargo build

# Create bootable disk images (UEFI + BIOS)
cargo run --manifest-path tools/image-builder/Cargo.toml -- \
    target/x86_64-unknown-none/debug/claudio-os

# Run in QEMU (BIOS mode, simplest)
qemu-system-x86_64 \
    -drive format=raw,file=target/x86_64-unknown-none/debug/claudio-os-bios.img \
    -serial stdio -m 512M

# With VirtIO networking (Phase 2+)
qemu-system-x86_64 \
    -bios /usr/share/OVMF/OVMF_CODE.fd \
    -drive format=raw,file=target/x86_64-unknown-none/debug/claudio-os-uefi.img \
    -device virtio-net-pci,netdev=net0 \
    -netdev user,id=net0 \
    -serial stdio -m 512M -smp 4
