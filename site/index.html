ClaudioOS

A bare-metal operating system for AI coding agents.
No Linux. No POSIX. No JavaScript.

Boots via UEFI on real hardware. Drives NVMe, AHCI, Intel NICs, USB 3.0 (keyboard + mouse), and NVIDIA GPUs from Ring 0. Authenticates to claude.ai via OAuth, runs multi-agent sessions with 6 pane types (agent, shell, browser, file manager, system monitor, screensaver) on a 2560x1600 framebuffer. Ships a post-quantum SSH daemon, inter-agent IPC, three language interpreters, 9 color themes, a text-mode web browser, session auto-refresh, and a boot splash with chime. Everything is #![no_std] Rust.

// Capabilities

Every driver, protocol, and runtime is written from scratch in #![no_std] Rust. No Linux kernel modules. No libc. No userspace.

Boot & Core

UEFI + SMP Multi-Core

UEFI boot with GDT/IDT, 48 MB heap, ACPI table parsing (MADT/FADT/HPET/MCFG), and symmetric multiprocessing. AP cores started via SIPI with work-stealing scheduler. Boot splash with progress bar and PC speaker chime.

Storage

NVMe + AHCI + SATA

Native NVMe driver with admin/IO queue pairs. AHCI/SATA for legacy disks. VFS layer with auto-detection across ext4, btrfs, NTFS, and FAT32 filesystems.

Networking

Intel NIC + TCP/IP + TLS 1.3

Drivers for e1000, I219-V, and I225-V Intel NICs plus VirtIO-net. smoltcp TCP/IP stack with DHCP and DNS. TLS 1.3 with AES-128-GCM from Ring 0. Intel NIC fully wired to smoltcp with DHCP.

Security

Post-Quantum SSH Server

ML-KEM-768 + X25519 hybrid key exchange. ML-DSA-65 host keys. SSH listener on port 22 with TCP session management. Up to 4 simultaneous sessions. User accounts with SHA-256 password and public key auth.

AI

Claude Multi-Agent Dashboard

OAuth device flow to claude.ai from bare metal. Max subscription integration. SSE streaming of Messages API. Split-pane terminal with 6 pane types. Inter-agent IPC with message bus, named channels, and shared memory. Session auto-refresh with JWT expiry detection.

Shell

AI-Native Shell

28 builtin commands plus natural language passthrough to Claude. Conversation management (list, select, rename, delete). Full pipeline support with the interpreters and editor.

USB

xHCI USB 3.0 + Mouse

Full xHCI host controller driver. USB keyboard events bridged to PS/2 scancode queue. USB HID mouse with XOR crosshair cursor on framebuffer. Device enumeration and endpoint configuration.

GPU

NVIDIA GPU Compute

BAR mapping, command submission, and compute dispatch on NVIDIA GPUs. MMIO register access from bare metal for GPU-accelerated workloads.

Audio

Intel HDA Audio + Boot Chime

High Definition Audio controller driver with codec discovery, widget parsing, and PCM stream output. PC speaker boot chime plays a C5-E5-G5 ascending triad at startup.

Languages

Python + JavaScript + Rust

Three interpreters/compilers running on bare metal. Python-lite (vars, loops, functions). JS-lite (ES5 core). Rust compiler via Cranelift JIT codegen. All no_std.

Browser

Web Browser Pane

Text-mode web browser as a dashboard pane. HTML parsing via wraith-dom, CSS selectors, HTTP/HTTPS transport over smoltcp. URL bar, link following with numbered references, history, and scroll navigation.

Files

Visual File Manager

Terminal-based file browser as a dashboard pane. Directory listing with color-coded types. Copy, move, rename, delete operations. Search/filter. VFS integration for all mounted filesystems.

Monitor

System Monitor

Real-time system stats in a dashboard pane. CPU usage with progress bar, memory (heap + framebuffer), network counters, per-agent state breakdown (thinking/streaming/tool/idle/error), and uptime. Auto-refreshes every second.

IPC

Inter-Agent Communication

Global message bus with per-agent inboxes. Named data channels (4K ring buffers). Shared memory regions. 8 IPC tools exposed to Claude agents for collaboration: send, receive, broadcast, channels, shared memory.

Themes

9 Color Themes

Runtime theme switching via theme <name>. Built-in: default, solarized-dark, solarized-light, monokai, dracula, nord, gruvbox, claudioos, templeos. ANSI 24-bit true color support.

Screensaver

5 Screensaver Modes

TempleOS-inspired screensavers: 3D starfield, Matrix rain, bouncing "ClaudioOS" logo (DVD style), randomly growing colored pipes, and a large digital clock. Activates after 5 minutes idle.

Init

Init System + Users

Boot configuration via fw_cfg (hostname, log level, auto-mount, SSH, startup scripts). User database with SHA-256 password hashing and SSH public key authentication. Default user with auto-login.

Boot

Boot Splash + Chime

ASCII art "CLAUDIOOS" logo centered on framebuffer with a 4-stage progress bar (Hardware, Network, Auth, Ready). PC speaker boot chime via PIT channel 2: C5 (523 Hz), E5 (659 Hz), G5 (784 Hz).

Editor

Nano-Like Text Editor

Buffer-gap text editor with syntax awareness. Integrated as the edit_file tool for Claude agents. ~400 lines, 11 tests.

// Built with Claude

ClaudioOS started with a question: What if an AI coding agent didn't need an operating system underneath it? What if it was the operating system?

Every line of this project was written in collaboration with Anthropic's Claude. The kernel boots, initializes hardware via ACPI, starts all CPU cores via SMP, authenticates to claude.ai, and spawns agent sessions that can write code, compile it (via Cranelift), execute Python and JavaScript, edit files, browse the web, manage conversations, communicate with each other via IPC, and SSH into remote machines -- all from bare metal with zero OS dependencies.

The development itself is recursive: Claude helps build the OS that runs Claude. The project has grown from a simple UEFI bootloader to a 33-crate workspace with 17 kernel modules and hardware drivers for NVMe, AHCI, Intel NICs, USB 3.0 (keyboard + mouse), HDA audio, and NVIDIA GPUs. It reads ext4, btrfs, and NTFS filesystems. It runs a post-quantum SSH daemon with user accounts. It has its own shell with natural language understanding, a web browser, a file manager, a system monitor, 9 color themes, 5 screensaver modes, and a boot splash with chime.

19 crates have been extracted and published as standalone open-source libraries, each usable in any #![no_std] Rust project. The goal isn't just an OS -- it's proving that AI-native systems programming is possible, practical, and productive.

// Build & Run

# Prerequisites
rustup target add x86_64-unknown-none
cargo install bootimage

# Build the kernel (33 crates + 17 kernel modules)
cargo build

# Create bootable disk images
cargo run --manifest-path tools/image-builder/Cargo.toml -- \
    target/x86_64-unknown-none/debug/claudio-os

# Run in QEMU with full networking + TLS
qemu-system-x86_64 \
    -bios /usr/share/OVMF/OVMF_CODE.fd \
    -drive format=raw,file=target/x86_64-unknown-none/debug/claudio-os-uefi.img \
    -device virtio-net-pci,netdev=net0 \
    -netdev user,id=net0,hostfwd=tcp::5555-:5555 \
    -serial stdio -m 512M -smp 4 -cpu Haswell

-cpu Haswell is required for AES-NI instructions used by TLS 1.3. For real hardware, the OS has been tested on i9-11900K, Supermicro SYS-4028GR-TRT, HP Victus laptop, and Arch Linux boxes.
